# Communications Checklist

**Purpose**: Guide the Communications Coordinator in managing stakeholder communications effectively throughout the project lifecycle.

**Owner**: Communications Coordinator

**Implements**: #4

## Stakeholder Mapping

### Identify Your Stakeholders

Complete this mapping at project kickoff and review quarterly or when major changes occur.

| Stakeholder | Role/Title | Interest Level | Influence Level | Communication Preference | Key Concerns |
|------------|-----------|----------------|-----------------|------------------------|--------------|
| | | ☐ High ☐ Medium ☐ Low | ☐ High ☐ Medium ☐ Low | ☐ Email ☐ Slack ☐ Meeting | |
| | | ☐ High ☐ Medium ☐ Low | ☐ High ☐ Medium ☐ Low | ☐ Email ☐ Slack ☐ Meeting | |
| | | ☐ High ☐ Medium ☐ Low | ☐ High ☐ Medium ☐ Low | ☐ Email ☐ Slack ☐ Meeting | |

### Stakeholder Segmentation

**High Interest + High Influence (Manage Closely)**:
- Frequent, detailed updates
- Early notification of issues
- Involvement in key decisions

**High Interest + Low Influence (Keep Informed)**:
- Regular status updates
- Access to detailed documentation
- Optional participation in reviews

**Low Interest + High Influence (Keep Satisfied)**:
- Executive summaries only
- Escalation notifications
- Milestone reviews

**Low Interest + Low Influence (Monitor)**:
- Occasional updates
- Included in general announcements
- On-demand access to information

## Communication Cadence

### Weekly Updates

**Audience**: Project Manager, Delivery Team, High-Interest Stakeholders

**Channel**: Email, Slack, or Project Dashboard

**Content**:
- [ ] Progress summary (what was completed)
- [ ] Upcoming work (next week's focus)
- [ ] Blockers and risks
- [ ] Key decisions made
- [ ] Action items and owners

**Template**: See "Weekly Status Update Template" below

---

### Bi-Weekly Executive Summaries

**Audience**: Project Sponsor, Executive Stakeholders

**Channel**: Email

**Content**:
- [ ] High-level progress (% complete, milestone status)
- [ ] Key achievements
- [ ] Top 3 risks and mitigation status
- [ ] Budget and timeline status
- [ ] Decisions needed from executives

**Template**: See "Executive Summary Template" below

---

### Sprint/Iteration Updates

**Audience**: Delivery Team, Product Manager, QA Lead

**Channel**: Slack, Team Meeting

**Content**:
- [ ] Sprint goals achieved
- [ ] Demo highlights
- [ ] Retrospective action items
- [ ] Next sprint preview

---

### Release Communications

**Audience**: All Stakeholders, Customers (if applicable)

**Channel**: Email, Announcement Blog, Internal Portal

**Content**:
- [ ] Release date and deployment window
- [ ] New features and improvements
- [ ] Known issues or limitations
- [ ] Support contact information
- [ ] Rollback plan summary

**Template**: See "Release Announcement Template" below

---

### Incident/Escalation Communications

**Audience**: Varies by severity (see escalation matrix)

**Channel**: Urgent: Phone/Slack, Follow-up: Email

**Content**:
- [ ] Issue description and impact
- [ ] Current status and timeline
- [ ] Mitigation actions in progress
- [ ] Next update time
- [ ] Contact for questions

**Template**: See "Escalation Messaging Template" below

## Escalation Messaging Guidelines

### When to Escalate

**Immediate Escalation** (within 15 minutes):
- Production outage or critical defect
- Security breach or data loss
- Major delay (> 2 weeks) to committed milestone
- Key team member departure
- Budget overrun > 20%

**Routine Escalation** (within 24 hours):
- Medium-priority risks materializing
- Scope change requests
- Resource constraint issues
- Minor delays (< 1 week)

### Escalation Recipients by Severity

**Critical Issues**:
- Project Sponsor (immediate)
- Executive Stakeholders (within 1 hour)
- All project team leads (immediate)

**High Priority Issues**:
- Project Manager (immediate)
- Project Sponsor (within 4 hours)
- Affected stakeholders (within 4 hours)

**Medium Priority Issues**:
- Project Manager (within 24 hours)
- Relevant team leads (within 24 hours)

## Communication Checklist

Use this checklist to ensure effective communication:

### Pre-Project
- [ ] Stakeholder mapping completed
- [ ] Communication cadence defined and agreed
- [ ] Templates customized for project
- [ ] Communication channels established
- [ ] Escalation paths documented and socialized

### During Project
- [ ] Weekly updates sent on schedule
- [ ] Executive summaries sent on schedule
- [ ] All stakeholders receiving appropriate level of detail
- [ ] Escalations handled per defined process
- [ ] Feedback loop established (stakeholders can ask questions)
- [ ] Communication effectiveness reviewed in retrospectives

### Pre-Release
- [ ] Release announcement drafted and reviewed
- [ ] Stakeholders notified of deployment window
- [ ] Customer-facing teams briefed
- [ ] Support team prepared with FAQs
- [ ] Escalation contacts confirmed

### Post-Release
- [ ] Release success/issues communicated
- [ ] Lessons learned documented
- [ ] Thank-you sent to team and stakeholders

| Release Announcements | Release Manager, Project Manager |
| Customer-Facing Communications | Project Sponsor, Product Manager |
| Escalation Messages (Critical) | Project Manager, Project Sponsor |
